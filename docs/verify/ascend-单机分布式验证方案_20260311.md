# 昇腾单机分布式验证方案 (vLLM-Ascend + MindIE)

> **验证日期**: 2026-03-11  
> **验证人**: zhanghui  
> **验证环境**: 7.6.52.170 (单机双 NPU) → k3s-in-Docker  
> **NPU**: 2 × 910B2C (davinci0 + davinci1, 各 65536 MB HBM)  
> **模型**: DeepSeek-R1-Distill-Qwen-1.5B  
> **引擎**: vLLM-Ascend (Ray) + MindIE (HCCL)  
> **项目**: infer-control-sidecar-unified  

---

## 一、验证目标

在 .170 单机上使用两张 910B2C NPU，通过 StatefulSet (replicas=2) 部署两个 Pod，分别验证：

1. **vLLM-Ascend Ray 分布式**: Master/Worker 角色协调 → Ray 集群 (HCCL) → vllm serve TP=2
2. **MindIE HCCL 分布式**: Master/Worker 角色协调 → config.json (worldSize=2) → mindieservice_daemon

覆盖代码路径：`main.py` → `distributed/master.py` / `worker.py` → `config_loader.py` → `vllm_adapter.py` / `mindie_adapter.py` → `proxy/` → 端到端推理

### 4.6 已知风险与排查

| # | 风险 | 排查方法 | 解法 |
|---|------|---------|------|
| 1 | Ray 无法感知 NPU | engine 日志: `ResourcesUnavailable` | vllm_adapter 中改 `--num-gpus=1` → `--resources='{"NPU":1}'` |
| 2 | HCCL 初始化失败 | engine 日志: `HCCL error` / timeout | 检查 `HCCL_WHITELIST_DISABLE=1`, `HCCL_IF_IP`, 两 Pod 间 TCP 连通 |
| 3 | CANN 环境缺失 | engine 日志: `libascendcl.so not found` | 确认 start_command.sh 中有 `source ...set_env.sh` |
| 4 | Pod IP 取不到 | wings 日志: `MASTER_IP=unknown` | 检查 /ip-exchange 目录权限，两 Pod 是否同时启动 |
| 5 | vllm-ascend TP 与 Ray worker 不兼容 | vllm 日志: PlacementGroup 错误 | 确认 TP=2, NNODES=2, 每 Pod DEVICE_COUNT=1 |

## 六、网络注意事项（关键）

基于 NV 验证积累的经验，以下网络问题在昇腾场景同样适用：

### 6.1 Pod 网络 vs hostNetwork

| 方案 | 单机分布式可行性 | 原因 |
|------|-----------------|------|
| hostNetwork | **不可行** ❌ | 同机两 Pod IP 相同 (7.6.52.170)，`_determine_role()` 无法区分角色 |
| Pod 网络 (CNI) | **可行** ✅ | 每 Pod 独立 IP (10.42.x.x)，互不冲突 |

### 6.2 CoreDNS 不可用

| 问题 | 原因 | 解法 |
|------|------|------|
| CoreDNS Pod 在 .110 (server)，.170 上的 Pod 无法查询 DNS | k3s-in-Docker 的 flannel 跨容器路由问题 | 使用 `dnsPolicy: Default` + hostPath IP 交换 |

### 6.3 hostPath IP 交换机制

```
Pod-0 启动 → 写 /ip-exchange/pod-0-ip → 等待 pod-1-ip
Pod-1 启动 → 写 /ip-exchange/pod-1-ip → 等待 pod-0-ip
                                       ↓
两个 Pod 2 秒内获得对方 IP → 注入 MASTER_IP / NODE_IPS
```

⚠️ **每次重新部署前务必清理** `/tmp/wings-ip-exchange/` 目录，避免残留旧 IP。

### 6.4 HCCL 通信 (昇腾特有)

| 变量 | 值 | 说明 |
|------|------|------|
| `HCCL_WHITELIST_DISABLE` | 1 | 容器环境必须禁用 HCCL 白名单 |
| `HCCL_IF_IP` | Pod IP (自动) | vllm_adapter 中通过 `$(hostname -i)` 获取 |
| `HCCL_SOCKET_IFNAME` | eth0 | Pod 网络 CNI 虚拟接口 |
| `hccn.conf` | 宿主机挂载 | MindIE 需要读取 HCCL 网络配置 |
